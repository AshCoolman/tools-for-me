## Specify - WHAT and WHY
- **Problem:** The repo uses Yarn 1 (classic), which has no supply-chain hardening controls. Migrating to pnpm v11 enables `minReleaseAge` (reject packages published within N hours — catches typosquatting and fast-moving compromised packages) and `noScripts` (prevent lifecycle scripts from executing arbitrary code at install time).
- **Outcome:** `packageManager` field set to `pnpm@11.x`, a `pnpm-workspace.yaml` covering the existing workspaces and pnpm project settings (`minimumReleaseAge`, `ignoreScripts`, `nodeLinker`), and `package-lock.json` replaced by `pnpm-lock.yaml`. CI and local `install` succeed.
- **Non-goals:** Changing any workspace's own dependencies or build config. Migrating off Lerna. Pinning specific package versions.
- **Success criterion:**
  1. `pnpm install` completes cleanly from a cold cache.
  2. `pnpm-workspace.yaml` contains `minimumReleaseAge: 20160` (14 days in minutes), `ignoreScripts: true`, `nodeLinker: hoisted`.
- **Threat-model link:** supply-chain / install-time code execution
- **Constraints:**
  - Must use pnpm v11 (not v9/v10).
  - `ignoreScripts: true` and `minimumReleaseAge: 20160` must be in `pnpm-workspace.yaml` (pnpm v11 moved project settings out of `.npmrc`; `.npmrc` is now auth/registry only).
  - `minimumReleaseAge`: 14 days = 20 160 minutes.
  - `node-linker=hoisted` (flat `node_modules`, compatible with Lerna and existing tooling).
  - Existing workspace glob patterns (`x/*`, `x/*/*`, `packages/*`) must be preserved in `pnpm-workspace.yaml`.
  - Do not commit `node_modules`.
  - Keep Lerna for cross-workspace orchestration.

## Plan - HOW

**Approach:** In-place migration — no branch required, all changes are root config files plus a new lockfile. No workspace package.json files need touching.

**Surface / files:**
| File | Change |
|---|---|
| `package.json` | `packageManager` → `pnpm@11.x`; `workspaces` removed; `resolutions` → `pnpm.overrides`; `scripts.test` `yarn` → `pnpm` |
| `pnpm-workspace.yaml` | New file; workspace globs + `minimumReleaseAge: 20160`, `ignoreScripts: true`, `nodeLinker: hoisted` |
| `.npmrc` | Auth/registry only — no hardening settings (v11 moved these to workspace yaml) |
| `pnpm-lock.yaml` | Generated by `pnpm install`; committed |
| `package-lock.json` | Delete (untracked, generated by accident) |

**Validation:**
1. `pnpm install` exits 0 from clean state.
2. `pnpm-workspace.yaml` contains `minimumReleaseAge: 20160`, `ignoreScripts: true`, `nodeLinker: hoisted`.
3. `pnpm run test` (Lerna path) exits without "yarn not found" or workspace errors.

**Backward-compat:** The `packageManager` field triggers Corepack to auto-install pnpm v11 for any contributor with Corepack enabled (`corepack enable` is a one-time step). No other tooling changes needed.

**Lock-in:** `pnpm-lock.yaml` is incompatible with Yarn/npm. Rollback = `git checkout` root files + `yarn install` to regenerate `yarn.lock`.

**Rollback:** Trivial — all changes are root-level config files with no downstream build consequences.

## Tasks

### T1 — Update `package.json`
- **Files:** `package.json`
- **Changes:**
  - `packageManager`: `"yarn@1.22.1"` → `"pnpm@11.1.3"`
  - Remove `workspaces` array (moved to `pnpm-workspace.yaml`)
  - Rename `resolutions` → `pnpm.overrides`
  - `scripts.test`: `"lerna exec -- yarn test"` → `"lerna exec -- pnpm test"`
- **Success:** `node -e "const p=require('./package.json'); console.log(p.packageManager, !!p['pnpm']?.overrides, !p.workspaces)"` prints `pnpm@11.1.3 true true`
- **Validation:** File parses as valid JSON; no `resolutions` key; no `workspaces` key
- **Budget:** 5 min

### T2 — Create `pnpm-workspace.yaml`
- **Files:** `pnpm-workspace.yaml` (new)
- **Changes:** New file with workspace globs and all pnpm project settings
  ```yaml
  packages:
    - 'x/*'
    - 'x/*/*'
    - 'packages/*'
  minimumReleaseAge: 20160
  ignoreScripts: true
  nodeLinker: hoisted
  ```
- **Success:** File exists and contains all three setting keys
- **Validation:** `grep -E 'minimumReleaseAge|ignoreScripts|nodeLinker' pnpm-workspace.yaml | wc -l` → `3`
- **Budget:** 5 min

### T3 — Generate lockfile and clean up
- **Files:** `pnpm-lock.yaml` (generated), `package-lock.json` (deleted)
- **Changes:** Run `pnpm install`; delete untracked `package-lock.json`
- **Success:** `pnpm-lock.yaml` exists; `package-lock.json` does not
- **Validation:** `pnpm install --frozen-lockfile` exits 0
- **Budget:** 10 min (network-dependent)

### T4 — Verify hardening settings are active
- **Files:** none (read-only check)
- **Validation:**
  1. `grep 'minimumReleaseAge: 20160' pnpm-workspace.yaml` exits 0
  2. `grep 'ignoreScripts: true' pnpm-workspace.yaml` exits 0
  3. `grep 'nodeLinker: hoisted' pnpm-workspace.yaml` exits 0
  4. `ls node_modules/.modules.yaml && grep 'hoistedDependencies' node_modules/.modules.yaml` exits 0 (confirms hoisted linker active)
- **Budget:** 2 min
